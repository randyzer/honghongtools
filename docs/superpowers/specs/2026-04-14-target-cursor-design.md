# Target Cursor Design

**Goal:** 为当前网站接入一个全站生效的自定义目标光标特效，保留原组件的旋转与吸附交互，同时视觉上适配网站现有的浅底粉紫风格。

**Chosen Direction**
- 全站生效
- 粉白适配版，而不是原始纯白版
- 桌面端启用，移动端和触屏设备自动禁用
- 自动识别站内常见交互元素，而不是要求逐页手动添加 `.cursor-target`

**Implementation Shape**
- 新增客户端组件 `TargetCursor`
- 在全局布局 `src/app/layout.tsx` 挂载一次
- 默认目标选择器覆盖 `a`、`button`、`[role="button"]`、可点击卡片等常见交互元素
- 通过 `globals.css` 增加少量主题变量，统一控制描边、中心点和发光颜色

**Behavior Rules**
- 桌面端默认隐藏系统光标
- 鼠标悬停交互元素时，四角吸附到目标边界
- 非交互区域保持基础旋转态
- 点击时中心点和外框有轻微缩放反馈

**Visual Adaptation**
- 角标和中心点使用粉白高对比色
- 保留柔和发光，避免在白底页面里看不清
- 不改变现有页面布局和组件结构

**Risks**
- 全站隐藏默认光标可能影响可用性
  - 预案：仅桌面端启用，且组件卸载时恢复原始 cursor
- 自动目标选择器过宽可能误吸附
  - 预案：首版只覆盖常见交互元素，不扫全站任意 `div`
- 新增 `gsap` 依赖
  - 预案：只用在该组件内，不扩散到其他动画逻辑

**Validation**
- 首页、博客列表页、博客详情页都能看到自定义光标
- Hover 到按钮、链接、卡片时吸附正常
- 移动端不渲染该组件
- 页面无新的 hydration 或 runtime 错误
